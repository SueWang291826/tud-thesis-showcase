ABOUT What Is This Project?

A complete research pipeline bridging BIM digital-twin technology with large language model agents.

Core Innovation This system takes raw IFC BIM files of a real subway station and — without any manual walkability annotation — automatically constructs a navigable graph, runs agent-based pedestrian simulations, and exposes everything to a conversational LLM through a typed tool SDK and dual-vector RAG retrieval.

🏗️ BIM → World Model

Parses three IFC files (Platform F1, Concourse F3, Transport F4) using ifcopenshell + Shapely to extract walkable polygons, obstacles, staircase/escalator/lift connectors, and platform screen doors.

🗺️ 2.5D Navigation Graph

0.5 m grid sampling produces 18,648 nodes and 68,148 edges across four floors. Cross-level connections model staircase, escalator, and elevator traversal with realistic time costs.

🚶 Agent-Based Simulation

200 Mesa agents traverse three scenarios (Static flow · Dynamic train arrival · Wheelchair accessibility) with Dijkstra routing, real-time congestion feedback, and route replanning.

🤖 Conversational LLM Agent

DeepSeek Function Calling + Regex fallback handles natural-language queries like "Which exit is least congested?" or "Plan a wheelchair route from platform to exit B." using RAG (ChromaDB) + FAISS node-vector index.

About This Thesis

This research investigates how BIM digital-twin infrastructure can be leveraged to power an LLM-based conversational navigation agent for complex multi-level transit environments. The station modelled is a real urban rail station with four floor levels, multiple connector types, and over 700 individual spatial elements.

The complete pipeline runs in 96.5 seconds on a standard laptop, producing a fully queryable navigation graph, three simulated pedestrian scenarios, and a deployed conversational agent.
